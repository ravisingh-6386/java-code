## 🚀 Getting Started

### Prerequisites

- **Java Development Kit (JDK)** 8 or higher
- **Java IDE** (IntelliJ IDEA, Eclipse, or VS Code with Java extension)
- **Git** for version control
- Command line/Terminal access

### Installation

1. **Clone the repository:**
   ```bash
   git clone https://github.com/ravisingh-6386/java-code.git
   cd java-code
   ```

2. **Verify Java installation:**
   ```bash
   java -version
   javac -version
   ```

3. **Open in IDE:**
   - IntelliJ: File → Open → Select project folder
   - Eclipse: File → Import → General → Existing Projects into Workspace
   - VS Code: Open Folder and install Java Extension Pack

### Compiling & Running Programs

#### Command Line Method:

```bash
# Compile a single file
javac array.java

# Run the compiled class
java array

# Compile all Java files
javac *.java

# Run specific class
java oop
```

#### Using IDE:
- Right-click on the file
- Select "Run As" → "Java Application"
- Or use keyboard shortcut (Ctrl+Shift+F10 in IntelliJ)

## 🎓 Essential Java Concepts

### Object-Oriented Programming (OOP)

**The Four Pillars:**
1. **Encapsulation** - Hide internal details, expose only necessary
2. **Inheritance** - Reuse code through parent-child relationship
3. **Polymorphism** - Same interface, different implementations
4. **Abstraction** - Define what, not how

### Collections Framework

**Commonly Used:**
- `ArrayList<T>` - Dynamic array implementation
- `LinkedList<T>` - Doubly-linked list implementation
- `HashSet<T>` - Unordered unique collection
- `HashMap<K, V>` - Key-value pairs with hashing
- `Stack<T>` - LIFO structure
- `Queue<T>` - FIFO structure

### Exception Handling

```java
try {
    // Code that might throw exception
} catch (Exception e) {
    // Handle exception
} finally {
    // Code that always executes
}
```

## 🐛 Troubleshooting

### Common Issues & Solutions

**Issue: "class not found" error**
```bash
# Ensure you're in correct directory
cd java-code
javac ClassName.java
java ClassName
```

**Issue: "public class declaration is expected"**
```java
// Solution: File name must match public class name
// File: MyClass.java
public class MyClass { }
```

**Issue: Scanner not working**
```java
// Make sure to import
import java.util.Scanner;

// Create Scanner object
Scanner sc = new Scanner(System.in);
```

**Issue: NoSuchElementException with Scanner**
```java
// Check if input is available before reading
if (sc.hasNextInt()) {
    int num = sc.nextInt();
}
```

## 📝 Best Practices

1. **Use Meaningful Names** - Variables, methods, classes
2. **Add Comments** - Explain complex logic
3. **Follow Naming Conventions**:
   - Classes: PascalCase (e.g., `MyClass`)
   - Methods/Variables: camelCase (e.g., `myVariable`)
   - Constants: UPPER_CASE (e.g., `MAX_SIZE`)
4. **DRY Principle** - Don't Repeat Yourself
5. **Single Responsibility** - One class, one purpose
6. **Write Unit Tests** - Verify your code works
